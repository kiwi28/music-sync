Data Flow — Downloading a Playlist

Step-by-step: Spotify playlist → MP3s in your library
1 You paste a Spotify URL in the React dashboard
2 React → POST to Node.js API → creates job in PocketBase
3 Background Worker picks up job: "download playlist X to folder Y"
4 spotdl fetches Spotify metadata → searches YouTube Music → downloads audio
5 Dedup check: worker compares against existing files + PocketBase download log
6 New tracks only → ffmpeg converts to MP3 → writes to /music/{folder}
7 Worker updates job progress in PocketBase → realtime push to React
8 Worker triggers Navidrome scan → music appears in all your clients

Key Design Choices

What this architecture gives you

  • Navidrome = zero custom streaming code
  • PocketBase handles auth, DB, and realtime in one binary
  • Worker survives browser close — jobs keep running
  • 3 containers, 1 docker-compose.yml, 1 VPS
  • GitHub Actions: SSH → pull → compose up
  • All open-source, all free

Trade-offs to be aware of

  • Navidrome doesn't have offline download — clients do (Symfonium, play:Sub support offline caching)
  • spotdl depends on YouTube Music availability
  • Single VPS = single point of failure (mitigated by Docker restart policies)
  • PocketBase is still evolving (but stable enough for this)
